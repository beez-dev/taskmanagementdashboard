# app/api/lib

Shared plumbing for every route handler in this API. Not a service layer — no business logic lives here. This is pure HTTP-plumbing: response formatting, error handling, the mock data store, and input validation helpers.

## Files

| File | Purpose |
|---|---|
| [`response.ts`](./response.ts) | `jsonOk`, `jsonError`, `ApiHttpError`, and the `errors` factory (`notFound`, `validation`, `badJson`). Every handler that needs to return or throw goes through these. |
| [`handler.ts`](./handler.ts) | `handle()` wrapper — catches `ApiHttpError` → formatted error envelope, unknown → 500. `readJson()` — parses the request body and throws `badJson` on malformed input. |
| [`store.ts`](./store.ts) | In-memory mock task store backed by `globalThis[Symbol.for("app.mock.taskStore")]` so it survives Next.js HMR module reloads. Seeded with 3 tasks on first access. Replace this with a real database client when moving off the mock. |
| [`validation.ts`](./validation.ts) | `parseCreateInput`, `parseUpdateInput`, `parseListParams`, `buildTask`. All throw `ApiHttpError(400)` on invalid input. Validation runs **before** resource lookup so malformed requests return 400 instead of 404. |

## Standard route pattern

Every handler is identical in structure:

```ts
export const VERB = handle(async (req, ctx) => {
  // 1. parse + validate input (400 on failure)
  const input = parseX(req);

  // 2. look up resource (404 on missing)
  const record = store.get(id);
  if (!record) throw errors.notFound("task", id);

  // 3. mutate + respond
  return jsonOk(result, { status? });
});
```

## Response envelope

All endpoints — including errors — return the same JSON shape:

```json
{
  "data": <T> | null,
  "error": { "code": "string", "message": "string", "details"?: {} } | null,
  "meta"?: { "total": number, "page": number, "pageSize": number }
}
```

`data` is non-null when `error` is null, and vice versa. `meta` is present on paginated list responses only.

## Status codes

| Code | When |
|---|---|
| 200 | GET detail, PATCH, DELETE |
| 201 | POST create |
| 400 | Validation failure, malformed JSON, empty PATCH body |
| 404 | Resource not found |
| 405 | Method not defined on route (auto-generated by Next.js) |
| 500 | Unhandled exception caught by `handle()` |
